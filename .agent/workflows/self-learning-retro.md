# /self-learning-retro

## Purpose

Turn a completed task into a bounded improvement without inventing new behavior from weak evidence.

## Procedure

1. Compare intended outcome, actual changes, validation results, and remaining risks.
2. Identify one or two reusable lessons.
3. Reject lessons based on isolated accidents unless they are safety-critical.
4. Propose a concrete rule, workflow, or test only when it would prevent recurrence.

## Output

- What worked.
- What failed or stayed risky.
- Candidate improvement with evidence and owner.

## Gate

No self-improvement may bypass validation or rollback requirements.
