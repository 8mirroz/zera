# /project-bootstrap-guided

## Purpose

Create or repair a project in a staged way: architecture first, dependencies second, runnable baseline third.

## Procedure

1. Confirm target directory, stack, package manager, and deployment assumptions.
2. Create only the minimum scaffold needed for the first runnable slice.
3. Install dependencies through the repo's preferred tooling.
4. Run the app or validation command before claiming success.

## Output

- Bootstrap decisions.
- Created files.
- First-run command and validation result.

## Gate

No scaffold is complete until it has a runnable smoke path.
