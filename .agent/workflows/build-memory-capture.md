# /build-memory-capture

## Purpose

Persist only reusable knowledge from completed work. Avoid dumping logs, chat transcripts, or one-off details.

## Procedure

1. Identify what changed and why it matters beyond the current task.
2. Extract stable commands, patterns, failure signatures, and rollback notes.
3. Write knowledge only to the approved docs or memory surface for the repo.
4. Exclude secrets, credentials, and speculative claims.

## Output

- Knowledge item or report update.
- Validation evidence.
- Follow-up queue, if a real unresolved issue remains.

## Gate

Memory capture must be factual, source-linked, and small.
