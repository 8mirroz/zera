# /kilo-mode-routing

## Purpose

Use strict modes to reduce confusion on complex tasks: architect, coder, debugger, reviewer.

## Procedure

1. Architect mode defines target and constraints.
2. Coder mode implements scoped changes.
3. Debugger mode reproduces and fixes failures.
4. Reviewer mode checks regressions and missing gates.

## Output

- Mode sequence.
- Mode handoff notes.
- Final validation result.

## Gate

Do not blend planning assumptions into implementation without recording them.
