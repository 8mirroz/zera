# Pattern: Prompt Automation Engine (PAE)
Date: 2026-04-14
Status: Validated
Scope: Platform Infrastructure

## 🏗️ Проблематика
При масштабировании системы до уровня Fractal Swarm (множество сабагентов), ручное управление системными промтами приводит к "дрифту" инструкций и потере детерминизма.

## 💡 Решение
Внедрение централизованной библиотеки промтов в формате YAML с программным доступом через Jinja2-движок.

## 🛠️ Архитектурные компоненты

### 1. Единый Реестр (`manifest.yaml`)
Все промты регистрируются с метаданными (ID, Role, Tier). Это позволяет выбирать промт на основе сложности задачи (C1-C5).

### 2. YAML-транспорт
Промты хранятся как YAML-объекты. Это позволяет инкапсулировать метаданные промта (версия, автор, рекомендуемая модель) вместе с его контентом.

### 3. Динамическая Инъекция (Jinja2)
Использование Jinja2 в `prompt_engine.py` позволяет безопасно подставлять контекст сессии (`{{goal}}`, `{{target_files}}`) в шаблоны промтов.

### 4. Рантайм-кеширование
Все отрендеренные промты сохраняются в `.agents/runtime/prompts/`.
**Зачем**: Это обеспечивает полную прозрачность — всегда можно проверить, с каким ИМЕННО промтом работал сабагент в конкретный момент.

## 📜 Команды управления
- **Рендеринг**: `python3 scripts/zera/prompt_engine.py --id <ID> --ctx '<JSON>'`
- **Просмотр ресурсов**: `ls configs/prompts/`

## ⚠️ Извлеченные уроки (Lessons Learned)
- **Формат MD vs YAML**: YAML предпочтительнее для инструментальной обработки, так как предотвращает ошибки парсинга при наличии сложной разметки внутри промта.
- **Внешние зависимости**: Проверка существования `jinja2` через `python3 -m pip show` должна предшествовать запуску движка.
