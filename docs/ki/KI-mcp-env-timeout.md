# Ошибка: контекстный дедлайн (context deadline exceeded) и ошибка переменных окружения при старте MCP

## Ошибка
При старте системы Antigravity, MCP сервера инициализировались с ошибками (в логах `context deadline exceeded` для npx серверов) и падали процессы (например, `stitch`, `google-developer-knowledge`, `supabase-mcp-server`) по причине отсутствия API ключей, так как переменные вида `${VAR}` не интерполировались из `.env`.

## Причина
1. `context deadline exceeded` возникала из-за того, что `npx -y` впервые скачивала, собирала и устанавливала Node модули для MCP серверов. Установка занимала больше времени, чем отведённый timeout, что приводило к сбросу подключения. 
2. Ошибки авторизации серверов, требующих API ключи (stitch, supabase), происходили из-за того, что в `mcp_config.json` они были прописаны как жесткие строки `${STITCH_API_KEY}`. JSON не резолвит их автоматически.

## Фикс
1. **Таймауты `npx`:** Пакеты (`@magicuidesign/mcp`, `@modelcontextprotocol/server-memory` и др.) установлены глобально через `npm install -g`. Теперь `npx` находит их мгновенно, тайм-аут больше не срабатывает.
2. **Переменные окружения:** Исправлен блок `args` в `mcp_config.json`. Добавлен `source /Users/user/.hermes/profiles/antigravity/.env 2>/dev/null || true;` в строку `bash -lc`. Литералевые JSON-значения ENV удалены. (Серверы, к которым нет ключей, как `google-developer-knowledge`, задизейблены `disabled: true`).

## Проверка
Скрипты теперь подгружают ключи напрямую из профиля Antigravity `.env`, а сервера загружаются мгновенно без перекачивания сторонних пакетов. Падение MCP серверов устранено.
