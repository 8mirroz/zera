# 🌌 Zera Prompt Library

Добро пожаловать в централизованное хранилище промтов системы Zera / Antigravity.

## Структура
- `/system` — Промты для ролей (Architect, Engineer, Reviewer, и т.д.).
- `/tasks` — Промты для конкретных рабочих процессов (Research, Audit, UI-Creation).
- `/templates` — Переиспользуемые фрагменты и форматы вывода.
- `/external` — Склонированные библиотеки-первоисточники.

## Реестр
Все промты должны быть зарегистрированы в [manifest.yaml](./manifest.yaml). Это позволяет системе выбирать промт на основе метаданных.

## Как использовать (для Агентов)
1. **Идентификация задачи**: Определите `task_type` или `role`.
2. **Поиск в манифесте**: Найдите подходящий `id` в `manifest.yaml`.
3. **Чтение и адаптация**: Считайте файл промта и адаптируйте его под текущий контекст (используя переменные или инъекцию данных).

## Правила создания "Ультра-промтов"
- **XML-структура**: Используйте теги `<thought>`, `<analysis>`, `<output>` для детерминизма.
- **Chain of Thought**: Обязательно требуйте пошагового рассуждения.
- **Инъекция контекста**: Явно указывайте места для динамических данных (напр., `{{target_files}}`).
- **Гарды (Guards)**: Всегда включайте негативные ограничения (что НЕЛЬЗЯ делать).
